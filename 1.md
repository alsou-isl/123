## Предусловия (preconditions)
1. Пользователь находится в модуле Параметры скважины

## Шаги тестирования
| Шаг |  Действие  |           ОР          | Статус |
|-----|------------|-----------------------|--------|
| 1 | Нажать на кнопку "Добавить" | Открывается окно "Добавление текущего забоя" |  |
| 2 | Нажать на кнопку Сохранить | В форме обязательные поля"Глубина забоя текущего", "Дата замера забоя текущего", "Способ отбивки забоя", "Состояние забоя" подсвечиваются красным цветом и выводится текст "Поле должно быть заполнено" |  |
| 3 | Ввести текст в поле "Характер аварийного оборудования" объемом более 2000 символов | Отображается ошибка "Ограничение: 2000 символов" |  |
| 4 | Ввести текст в поле "Примечание" объемом более 2000 символов | Отображается ошибка "Ограничение: 2000 символов" |  |
| 5 | Ввести отрицательное число в поле "Глубина текущего забоя" | Отображается ошибка "Значение должно быть больше/равно 0" |  |
| 6 | Ввести в поле "Глубина текущего забоя" число, превышающее допустимые значения | Отображается ошибка "Значение должно быть не больше, чем 10000" |  |
| 7 | Ввести в поле "Глубина текущего забоя" значение меньше значения глубины начала забуривания | Отображается ошибка "Не может быть меньше, чем Глубина начала забуривания" |  |
| 8 | Ввести в поле "Дата забоя текущего" будущую дату | Отображается ошибка "Нельзя вводить будущую дату" |  |
| 9 | Ввести в поле "Дата забоя текущего" дату, которая была введена в другом забое | Отображается ошибка "На данную дату уже создана запись о текущем забое" |  |
| 10 | Ввести в поле "Дата аварии" будущую дату | Отображается ошибка "Нельзя вводить будущую дату" |  |
| 11 | Ввести в поле "Дата аварии" некорректное значение, нажать на кнопку Сохранить | Введенное значение сбрасывается |  |
